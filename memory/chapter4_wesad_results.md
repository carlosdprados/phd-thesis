Chapter 4 WESAD demonstrations were run on the real dataset on 2026-06-04 (15 subjects S2–S17; `data/wesad/`, gitignored). Canonical results + narrative are in [`handouts/12_chapter4_demonstration_plan_v4.md`](handouts/12_chapter4_demonstration_plan_v4.md) **§13**; scripts are `scripts/ch4_{model,reservoir,wesad,figures}.py`; figure `figures/chapter4/wesad_affect.pdf`.

**Key outcome (do not re-attempt the old "heterogeneity wins on affect" framing):**
- Demo A (window binary stress/baseline, EDA, lead bank): macro-F1 **0.894** — but a static baseline ties it (0.899), so the windowed task is quasi-static.
- Demo B was rebuilt as a **streaming** per-step affect-tracking task (the memory-demanding fair test, chosen by the author). Decomposition: instantaneous 0.709 → +dimensionality 0.737 → +memory ~0.753 → +heterogeneity 0.758. **Timescale heterogeneity is within noise: het−hom = +0.005 ± 0.011 (7/10 seeds).** Genuine fading-memory gain ≈ +0.014–0.020.
- **Heterogeneity is a computational resource only at the architecture/benchmark level** (Memory Capacity het 6.12 vs hom 4.10 = 1.49×; NARMA/MC rank the lead PEO 0.3/0.09 top). On affect the discriminative memory demand is dominated by one slow tonic band the lead composition already serves.

**Why:** affect labels are slowly-varying sustained states (single dominant timescale), so a spread of timescales doesn't convert to task F1. This unifies Ch3→Ch4: composition/drive *set* the timescale; Ch4 *matches* it to the task; heterogeneity buys task-agnostic memory breadth, not a guaranteed application win.

**How to apply:** when drafting `chapters/chapter4_temporal.tex`, frame it as (benchmarks = where heterogeneity wins) + (WESAD = devices do real affect, memory helps, heterogeneity is an honest reported null). **Jury-strengthening pass 2026-06-09** (Ch5 = `chapters/chapter5_temporal.tex`; the old null is now *reframed*, not removed). The key move: the "static baseline ties Demo A" weakness is overturned by testing what averages hide — **deployment robustness**, where fading **memory** (not its heterogeneity) is decisive.
- **Continuous stress monitoring under sensor noise** (`scripts/ch5_onset.py`, fig `robust_monitoring.pdf`, `sec:ch5_robust`): binary stress-vs-(baseline+amusement), per-step LOSO, linear read-out, + injected Gaussian/motion-burst noise. Clean ≈0.92 all banks (static competitive); under noise the instantaneous/memoryless controls collapse (0.914→0.840 at σ=0.4) while memory banks stay flat (≈0.92) → **het−inst +0.088, 15/15 subjects, p<1e-4**; memory ≈ halves false alarms (0.046→0.026). hom≈het (memory is the ingredient, not its spread). Latency comparable (smoothing-set); transition-F1 positive but not resolved.
- **Wrist + operating budget** (`scripts/ch5_deployment.py`, `sec:ch5_deploy`, `tab:ch5_wrist` + SI `tab:ch5_deployment`): WESAD Empatica-E4 wrist (EDA+Temp+PPG-HR; loaders in ch5_wesad.py) — static drops (0.914→0.881) but memory holds (0.920→0.922); advantage LARGER on wrist. Envelope from measured Ch2 device (50 nJ/event, 6 fJ/µm², 0.0825 cm², 0.5 V sub-threshold read): **0 trained dynamics params + 75–147 read-out weights (one ridge solve) vs 1e4–1e6 deep; ≈1–2 µW always-on**; areal cross-check 49.5≈50 nJ.
- **No-regret reframe** (`tab:ch5_summary`, intro+summary rewritten): memory = first-order resource; **heterogeneity = low-cost generality hedge** — wins multi-τ tasks (physio R², MC), the single best-matched composition wins single-τ tasks (sustained labels, monitoring), gap ≤0.01. The original label-heterogeneity null stands.
- **Cross-corpus replication** (`scripts/ch5_noneeg.py`, `tab:ch5_crosscorpus`): PhysioNet **Non-EEG** corpus (Birjandtalab 2016; 20 subj, Affectiva-Q wrist EDA+Temp + pulse-ox HR; `wfdb.dl_database('noneeg','data/noneeg')`, gitignored).
